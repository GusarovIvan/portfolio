# Портфолио — Гусаров Иван

Редакционный сайт-портфолио фотографа. HTML, CSS и JavaScript.

## Запуск

Сайт загружает JSON через `fetch`, поэтому нужен локальный сервер:

```bash
python -m http.server 8080
```

Откройте http://localhost:8080

## Как добавить новую съёмку

1. Положите фото в `photos/` — в существующую папку категории или создайте новую.
2. Создайте файл `data/projects/ваш-id.json`:

```json
{
  "id": "conf-new-event",
  "category": "conferences",
  "title": "Название мероприятия",
  "date": "01.01.2026",
  "shortDescription": "Краткое описание для ленты",
  "photoFolder": "конференции",
  "photoPrefix": "название_файла",
  "location": "Город",
  "description": [
    "Первый абзац для журнального вида.",
    "Второй абзац."
  ]
}
```

3. Запустите скрипт обновления:

```bash
python scripts/update-manifest.py
```

Скрипт автоматически найдёт фото по `photoFolder` + `photoPrefix` и обновит `manifest.json`.

### Категории (`category`)

| Значение       | Фильтр на сайте                    |
|----------------|------------------------------------|
| `conferences`  | Конференции                        |
| `student`      | Студенческие мероприятия           |
| `wedding`      | Свадьба                            |
| `concerts`     | Концерты                           |
| `phototour`    | Фототур с Викторией и Сергеем      |

### Именование фото

Если `photoPrefix` = `"масленица"`, подойдут файлы: `масленица.jpg`, `масленица (2).jpg` и т.д.

Если `photoPrefix` пустой — берутся все изображения из папки.

## Отзывы

Редактируйте `data/reviews.json`. Ссылку на форму замените в поле `formUrl`.

## Структура

```
photos/           — фотографии по категориям
data/projects/    — JSON каждой съёмки по id
data/manifest.json — список для ленты (генерируется)
data/reviews.json  — отзывы
scripts/          — update-manifest.py
```
